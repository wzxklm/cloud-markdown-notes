# 可测试功能

本文档是全功能测试覆盖清单。新增、删除或修改功能时，先更新本文件，再同步更新 API、CLI、Web 三端测试脚本，确保 `npm test` 始终覆盖当前完整功能面。

测试入口：

- API: `tests/api/full-test.ts`
- CLI: `tests/cli/full-test.ts`
- Web: `tests/web/*.spec.ts`
- 总入口：`tests/full-test-runner.ts`

## 维护规则

- 每个用户可感知功能都应在清单中有稳定功能编号。
- API、CLI、Web 任一端不支持某功能时，在对应列写 `不适用`，不要留空。
- 新增功能时，应同时补充成功路径、权限/认证、输入校验和关键失败路径。
- 修改功能行为时，应更新“必测断言”，并检查三端测试是否仍对应。
- 删除功能时，应删除或标记清单项，并同步删除测试脚本中的旧断言。
